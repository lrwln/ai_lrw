# 无状态 stateless
- 调用llm 接口的本质是什么？
http 调用 消耗算力 生成结果
高并发、高可用....该怎么办呢---后端需要支持无状态stateless
- 无状态是什么？
  http 无状态协议(无状态 get/post...restful) + header(cookie 身份)
  - 无状态是指每次请求都是独立的，服务器不会保存客户端的状态信息。每次请求都必须包含所有必要的信息，以便服务器能够处理该请求。
  - 无状态的好处：
    - 可扩展性：服务器可以轻松地处理大量请求，因为它不需要维护客户端的状态。
    - 简化设计：无状态的系统通常更容易设计和实现，因为它们不需要处理复杂的状态管理。
    - 容错性：如果一个服务器实例失败，其他实例可以继续处理请求，而不会丢失任何状态信息。
- 有状态？ 服务器压力太大了
  - 有状态是指服务器会保存客户端的状态信息，每次请求可能依赖于之前的请求。
  - 有状态的缺点：
    - 扩展性差：服务器需要维护客户端的状态，增加了复杂性和资源消耗。
    - 容错性差：如果一个服务器实例失败，可能会导致状态丢失，影响用户体验。

- llm 无状态 it基于http协议
  - 每个请求独立
