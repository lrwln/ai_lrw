## Token 词元
用户输入的Prompt (自然语言) 分割为多个token ,每个token 转成一个tokenId, 这些tokenId 就是模型的输入。
编码器 将 词编码为tokenId   decode解码器 好返回给我们 

大模型处理的最小单元不一定是完整的“词”， 可能是“子词”， “字符” 甚至 “标点”。用词元 能更精准地表达它是模型计算和计费的通用最小处理单位，而非语言学意义上的词。有利于模型计算，
unhappiness 词   词元？ un   happi  ness 
llm 通常不会把它当成一个完整的词， 而是将其分解为多个词元。
tokenization token ization

计算量太大了。把词切成字词， 模型只需要掌握几万个基础积木(词元)

token查找表  size 就不会那么大，运算效率高。

## Embedding 
语义向量 pre-trained 神经元
第一步： 用tokenId（数字表达） 变成了一个Embedding（语义向量） 坐标
你 -> tokenId (57668)  没有任何含义
不能通过这个编号， 加减乘除计算得到好的编号 ，
你-> 好  高维向量空间中距离比较近 ？
语义 + 距离计算 
向量是多维空间里的一个坐标点， 向量是有方向， 计算距离
语义相近的词， 他们的向量距离就比较近。

第二步  查找embedding (1024)， 计算**语义距离**
llm  把编号查表转成一个高维的向量， 这个过程就叫embedding.
llm 内部有个巨大的向量查找表  Embedding Matrix

模型拿到 Token ID 57668 “你“ 就直接去 57668 的柜子里， 把对应的向量”抽出来“

国王 王后 这两个点的距离， 接近
国王 - 男性的向量 + 女性的向量 = 王后  向量的计算和语义的迁移

苹果 很远
模型的训练就是会构建这样的几何结构和空间坐标系。

## 位置编码
语义的特例之一
”我咬了狗“
”狗咬了我“
光有语义的理解还不够， 相同的4个字， 顺序不一样， 表达的意思不一样。
顺序 当前上下文下非常重要 
embedding 不携带位置信息， 我们就给每个向量叠加一个位置编码（
Position Encoding PE）, 告诉llm, 这个词属于句子的第几个。

每个token 携带两类信息： 语义信息（是什么）， 位置信息（在哪里）
