# RAG 

## llm 的幻觉
大模型知道的知识， 取决于训练时给它的**数据集**。
如果你问它最近发生的事情，或者你企业内部私有的文档，llm是不知道。
但它不会说自己不知道，而是会胡乱回答， 也就是所谓的幻觉。 

## 如何解决大模型的幻觉？
想一下

微调finetue  成本有点高  大公司、专业领域， 算力要求。

用户要查询的内容， 我们先去内部**知识库**里查一下， 把它放到Prompt 里再给大模型。

大模型通过这些**知识库**里的文档知道了背景知识，就可以回答问题。

RAG 
Retrieval 检索 - Augmented 增强  Generation 生成

去知识库里**检索**用户问的知识的相关文档片段(分片， 按章节， 按页， 按段落 一段段有自然语义的段落)（embedding）， 作为背景知识加到Prompt 里**增强**它，让大模型根据这些生成**回答**

- 用户问了一个问题， 你怎么把相关的文档片段查出来？

-  关键词搜索？ 
  文本匹配， 不太行
- 向量 vector 语义查询就可以 

  食用性、 0=无  -  1 高
  
  硬度    0 液体 -   1 硬

  水果   [0.9, 0.3]
  苹果   [0.9, 0.5] 
  香蕉   [0.9, 0.1]
  石头   [0.1, 0.9]

  多维向量来表达丰富的语义， 苹果， 水果 香蕉 相关性很大， 石头不大。
  可以通过夹角， 越小越相似。 余弦。


  RAG 一般是结合向量化来做， 比关键词搜索， 更牛。

  用到专门的模型， 叫嵌入模型（Embedding Model）
  便宜一点， 和Generative Model 不一样 。
  可以把知识库里的文本、图片、pdf、语音， 向量化后，都可以语义搜索。

## 两个突破
  - 文档怎么切分
  - 向量 怎么把相关的文档查出来？
  - 向量数据库？
    - 数据库存向量， mysql 等传统的做不到。
    - 存的很多， 持久化存储 

在原始prompt 给到大模型之前， 查询下知识库，把相关的文档作为背景知识加入到Prompt里， 再让大模型回答， 这就是 RAG。

RAG 要实现语义查询， 需要基于向量来做， 把文档向量化存储到**向量数据库**， 查询的时候也把Prompt 向量化， 去数据库中做相识度检索， 可以找到语义相近的文本块。
